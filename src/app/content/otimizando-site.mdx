# Dicas para otimização de sites

Resumo de práticas mencionadas por Sérgio Lopes em entrevista sobre performance web.

## Checklist

1. Ative compressão com Gzip ou Brotli.
2. Otimize imagens antes de publicar.
3. Prefira SVG quando fizer sentido.
4. Use sprites ou `symbol` em SVG para reaproveitamento.
5. Mantenha o HTML inicial o mais enxuto possível.
6. Reduza o número de requisições.
7. Minifique CSS, JavaScript e HTML.
8. Lembre que o navegador paraleliza um número limitado de requests por origem.
9. Aproveite carregamento paralelo quando possível.
10. Configure `Cache-Control` corretamente.
11. Avalie recursos inline com critério.
12. Use [WebPageTest](http://webpagetest.org) para medir.
13. Revise o caminho crítico de renderização.
14. Atrase JavaScript não essencial.
15. Use lazy load para imagens e vídeos.
16. Priorize conteúdo *above the fold*.
17. Aplique técnicas de *throttle* quando necessário.
18. Remova listeners quando não forem mais necessários.
19. Considere CSS crítico inline no topo.
20. Avalie os ganhos reais de HTTP/2 e estratégias modernas de entrega.
21. Use *resource hints* com parcimônia.
22. `preconnect`
23. `dns-prefetch`
24. `prefetch`
25. `preload`
26. Defina um orçamento de performance com [performancebudget.io](http://performancebudget.io).
