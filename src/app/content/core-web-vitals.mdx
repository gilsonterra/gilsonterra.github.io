# Core Web Vitals

Os Core Web Vitals são métricas acompanhadas pelo Google para avaliar a experiência real de uso de uma página. Elas ajudam a identificar gargalos de carregamento, interatividade e estabilidade visual.

## LCP — Largest Contentful Paint

O **LCP** mede a percepção de carregamento da página. Em uma boa experiência, o maior elemento visível deve aparecer em até **2,5 segundos** após o início do carregamento.

### Como melhorar o LCP

- Use CDN próxima do usuário.
- Adote estratégias de cache.
- Comprima arquivos com Gzip ou Brotli.
- Carregue imagens de forma otimizada.
- Use `preconnect`, `dns-prefetch` e `preload` quando fizer sentido.
- Minifique CSS, JavaScript e imagens.
- Extraia ou inlinize o CSS crítico.
- Adie recursos não críticos.

Referência: https://web.dev/optimize-lcp/

### Métricas relacionadas ao LCP

#### TTFB — Time to First Byte

Tempo até o primeiro byte da resposta chegar ao navegador.

#### FCP — First Contentful Paint

Momento em que o primeiro conteúdo perceptível é renderizado na tela.

### Ações úteis para FCP e TTFB

- Escolha bem a infraestrutura e a região do servidor.
- Otimize a compressão de arquivos e assets.
- Comprima imagens antes do upload.
- Revise o impacto de fontes, scripts e folhas de estilo bloqueantes.

Ferramentas úteis:

- https://tinypng.com
- https://jakearchibald.github.io/svgomg

## FID — First Input Delay

O **FID** mede o atraso entre a primeira interação do usuário e o momento em que o navegador consegue responder. O ideal é ficar em até **100 milissegundos**.

### Como melhorar o FID

- Quebre tarefas longas de JavaScript.
- Reduza o custo de scripts de terceiros.
- Avalie renderização no servidor quando fizer sentido.
- Use `defer` e `async` em scripts não críticos.
- Considere Web Workers para trabalho pesado fora da thread principal.

Referência: https://web.dev/optimize-fid/

### Métricas relacionadas ao FID

#### TBT — Total Blocking Time

É uma métrica de laboratório usada como aproximação quando não há interação real do usuário para medir o FID.

#### TTI — Time to Interactive

Mede quanto tempo a página leva para se tornar utilizável de forma confiável.

## CLS — Cumulative Layout Shift

O **CLS** mede a estabilidade visual da interface. O valor ideal é de até **0,1**.

### Como melhorar o CLS

- Defina largura e altura para imagens e mídia.
- Reserve espaço para anúncios, embeds e iframes.
- Evite inserir conteúdo acima do que já foi renderizado.
- Garanta que o layout responsivo não provoque saltos inesperados.

Referência: https://web.dev/optimize-cls/
